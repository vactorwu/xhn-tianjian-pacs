  微狗和USB狗 Windows 9x//ME/NT/2K 驱动安装程序是安装在 Windows 9x/NT/2K以及
  Windows ME环境下的驱动的程序。

  加密后的 Win32 应用程序在 Windows 9x以及Windows ME 下运行时，需要安装硬件
  狗Windows 9x 的设备驱动程序；在未装驱动的情况下也可正常运行，但在打印时同
  时操作狗可能影响打印。

  加密后的 DOS16，WIN16，WIN32 应用程序在 Windows NT/2K 下运行时，必须要安装
  硬件狗 Windows NT/2K 的设备驱动程序。

  运行INSTDRV.EXE按其提示，按‘继续’键继续安装，该软件可自动识别当前环境，
  并提示安装状态。

  软件开发商不但在加密时需要安装该设备驱动程序，而且在发行软件时，也应根据
  最终用户的操作系统，安装相应的设备驱动程序。

  如果您需要卸载狗的驱动程序，请在命令行方式下运行UNINSDRV.EXE。该软件可自
  动识别当前环境，完成驱动程序的卸载工作。

附录: A
  Dogsetup.dll是WIN32下动态连接库,开发商可用于为其加密的软件安装程序中.
  调用函数InstDriver, 在源文件中定义如下:

  int (WINAPI *InstDriver)( int ntd, int verold, char * currentDir);

  int ntd: 该动态连接库在Windows NT下,如果要安装16位DOS支持模块赋值为1, 
           不安装16位DOS支持模块则赋值为0。
  int verold: 为保留参数，可任意赋值。
  char * currentDir: 安装时驱动程序所在目录。如果是当前目录，赋值为“.”即可。

  如果运行成功则函数返回0，否则返回错误值。

  在本目录下的VCSAMPLE目录有一用VC编写的调用Dogsetup.dll实现驱动程序安装的
  例子。您可以参考此例子编写您的安装程序。

  我们还提供了一个安装驱动程序的工具Doginst.exe，这个工具调用Dogsetup.dll来实现
  安装RG-UMH的驱动程序。这个工具还要用到一个叫做Showsign.ini的配置文件。如果
  Showsign.ini文件中的值为TRUE，则Doginst.exe显示安装界面。如果Showsign.ini文件中
  的值为FALSE，将不显示安装界面。开发商如果不希望在您的安装程序中出现彩虹天
  地公司的字样，您可以在您的程序中调用Doginst.exe来安装狗的驱动程序。但如果您是
  在Windows 9x平台上调用Doginst.exe，则您必须重新启动机器，才能完成整个驱动程序
  的安装工作。在重新启动机器前，请不要执行任何对狗的操作，否则可能会造成系统不
  稳定。
